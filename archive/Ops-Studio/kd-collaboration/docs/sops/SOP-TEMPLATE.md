## Instructions for Using This Template

**When creating a new SOP:**

1. **Copy this template** to appropriate directory:
   ```bash
   cp docs/sops/SOP-TEMPLATE.md docs/sops/[category]/[your-sop-name].md
   ```

2. **Replace all bracketed placeholders** with actual content

3. **Delete sections you don't need** (Notes & Context, Related SOPs if none exist)

4. **Be specific and actionable** - someone unfamiliar should be able to follow

5. **Add to index:** Update `docs/sops/README.md` with entry in appropriate table

6. **Test it:** Follow your own SOP to verify it works

7. **Commit to git:**
   ```bash
   git add docs/sops/
   git commit -m "docs: Add SOP for [process name]"
   ```

**SOP Writing Best Practices:**

- **Use active voice:** "Run the command" not "The command should be run"
- **Be explicit:** Don't assume knowledge - spell it out
- **Include examples:** Show, don't just tell
- **Anticipate problems:** Add troubleshooting for common issues
- **Make it scannable:** Use headers, bullets, checklists
- **Define success:** Clear criteria for "done"
- **Keep it current:** Update when process changes
